## 二、阶段一：UI 视觉升级

### 2.1 当前问题

- 主色调单一（只有 `#ff6b6b`），缺少层次感
- 整体风格偏保守，脱口秀应有的"轻松、幽默、年轻"感不足
- 前台（浅色红）与后台（深色侧边栏 `#2c3e50`）风格割裂，不像同一产品
- 移动端导航栏在窄屏下堆叠混乱
- 空状态和错误提示只有纯文字，缺少视觉反馈
- 卡片交互只有 hover 上移，缺少微动效细节

### 2.2 改造方案

#### 2.2.1 色彩体系重构

建立统一的品牌色板（脱口秀风格——年轻、活力）：

- **主色**：暖橙 `#FF6B35` → 过渡到活力紫 `#9B59B6`（渐变）
- **辅色**：亮黄 `#F7C948`（用于标签、促销标识）
- **背景**：暖灰 `#FDF8F5`（比纯白更温暖）
- **强调**：保持红色 `#FF4757` 用于紧迫感（已售罄、倒计时）
- **文字**：深灰 `#2D3436` 替代纯黑，次级文字 `#636E72`

#### 2.2.2 前台风格改造

- 导航栏：半透明毛玻璃效果（`backdrop-filter: blur`），页面滚动时粘性固定
- 演出卡片：增加阴影深度分层、图片 hover 放大带光晕、标签圆角胶囊化
- 文字排版：标题使用更粗的字重，票价使用渐变色数字
- 空状态：替换为带插画的 SVG 空状态组件（无演出、无订单、无退票）
- 按钮：统一圆角，增加点击波纹效果

#### 2.2.3 后台风格统一

- 侧边栏改为浅色/白色背景，与前台品牌色统一
- 统计卡片增加微图标动画
- 表格行 hover 效果优化

#### 2.2.4 移动端适配

- 导航栏改为汉堡菜单（手机端）
- 演出列表在手机上改为单列布局，卡片宽度撑满
- 表格（订单、后台）增加横向滚动提示

## 三、阶段二：功能增强

### 3.1 抢票倒计时

- 在演出卡片上，如果距离开场/开抢时间 < 3 天，显示实时倒计时（天/时/分/秒）
- 倒计时归零后自动刷新页面状态
- 使用 JS `setInterval` 客户端实现，避免服务器轮询

### 3.2 演出详情页

- 新建 `show-detail.jsp`，独立于现有 `show-detail` 接口
- 展示：大图海报、完整介绍、演员介绍/照片、演出时长、场馆地图/介绍
- 底部固定购票按钮

### 3.3 模拟支付弹窗

- 替代现有直接"购票成功"的流程
- 弹窗展示：订单摘要、支付方式选择（微信支付/支付宝/模拟支付）
- 选择后展示"支付中"动画 → 支付成功/失败结果
- 增加购票成功动画（彩带/对勾弹出）

### 3.4 订单详情页

- 点击订单号可查看详情
- 展示：演出海报缩略图、订单号、观众信息、座位信息、支付时间、状态流转

### 3.5 热门演出排行

- 根据售票速度（`(totalTickets - availableTickets) / daysSinceListed`）计算热度
- 首页增加"🔥 热门排行"侧边栏或板块

### 3.6 验票核销功能（管理员端）

- 订单增加核销状态（unused/used）
- 管理员可输入订单号或扫描二维码核销入场
- 核销后不可退票

### 3.7 演出评价功能

- 购票并观看完演出后可评价（评分 1-5 星 + 文字）
- 演出详情页展示评价列表

## 四、阶段三：交互体验优化

### 4.1 Loading 状态

- 所有 AJAX 请求期间，按钮显示 spinner 动画
- 页面加载时，卡片区域显示骨架屏（skeleton screen）

### 4.2 Toast 通知

- 将 `alert()` 弹窗全部替换为底部 Toast 通知
- 成功绿色、错误红色、警告黄色、信息蓝色
- 自动 3 秒消失，可手动关闭

### 4.3 表单验证增强

| 字段 | 验证规则 |
|------|----------|
| 手机号 | 11 位数字，以 1 开头 |
| 邮箱 | 标准邮箱格式 |
| 学号 | 根据学校格式验证 |
| 姓名 | 2-20 个字符，不允许空 |

### 4.4 微交互动效

- 卡片列表：渐入加载动画（stagger）
- 筛选切换：平滑过渡，而非瞬间显示/隐藏
- 购票成功：全屏遮罩+成功动画+自动跳转
- 页面切换：淡入淡出效果

## 五、阶段四：代码结构优化

### 5.1 资源文件分离

```
webapp/
├── css/
│   ├── main.css          # 前台主样式
│   ├── admin.css          # 后台样式
│   └── animations.css     # 动画合集
├── js/
│   ├── main.js            # 通用函数
│   ├── shows.js           # 演出列表页逻辑
│   ├── order.js           # 订单页逻辑
│   └── admin.js           # 后台逻辑
└── images/                # 静态图片资源
```

### 5.2 JSP 抽公共模板

- 将导航栏、页脚、侧边栏抽为独立 JSP 片段（`<jsp:include>`）
- 统一页面布局模板，减少重复代码

### 5.3 前端组件化（可选）

- 如后续开发量大，可引入 Vue 3 CDN + 单文件组件模式
- 对核心交互（筛选、搜索、购票流程）做组件化封装

## 六、实施路径建议

### 推荐顺序

```
阶段一 (UI 升级) → 阶段三 (交互优化) → 阶段二 (功能增强) → 阶段四 (代码整理)
```

### 原因

1. **UI 优先** — 视觉改造独立于业务逻辑，改动风险低，效果最明显
2. **交互其次** — 不影响功能的前提下提升体验
3. **功能再议** — 新功能较多，可根据需求选择性实现
4. **代码最后** — 重构在功能稳定后进行，避免重复工作

### 工作量估算

| 阶段 | 预估时间 | 涉及文件数 |
|------|----------|------------|
| 阶段一：UI | 2-3 天 | ~10 JSP + CSS |
| 阶段二：功能（选做） | 3-5 天/项 | ~5-8 文件/项 |
| 阶段三：交互 | 1-2 天 | ~6 JSP + JS |
| 阶段四：代码 | 1-2 天 | ~15 文件 |
